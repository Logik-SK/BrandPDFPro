BrandPDF Pro - EXE Packaging Process
Prerequisites
1. Install JDK 21

Verify:

java -version

Expected:

openjdk version "21.x"
2. Configure JAVA_HOME

Verify:

echo %JAVA_HOME%

Expected:

C:\Program Files\Java\jdk-21.x.x
3. Verify jpackage

Verify:

jpackage --version

Expected:

21.x.x
4. Install Maven

Verify:

mvn -version
pom.xml Configuration

Need:

JavaFX Plugin
<plugin>
    <groupId>org.openjfx</groupId>
    <artifactId>javafx-maven-plugin</artifactId>
    <version>0.0.8</version>
    <configuration>
        <mainClass>
            com.brandpdfpro.app.BrandPDFProApp
        </mainClass>
    </configuration>
</plugin>
Maven Shade Plugin

Required for fat JAR:

<plugin>
    <groupId>org.apache.maven.plugins</groupId>
    <artifactId>maven-shade-plugin</artifactId>
    <version>3.5.1</version>

    <executions>
        <execution>
            <phase>package</phase>

            <goals>
                <goal>shade</goal>
            </goals>

            <configuration>

                <createDependencyReducedPom>false</createDependencyReducedPom>

                <transformers>

                    <transformer implementation="org.apache.maven.plugins.shade.resource.ManifestResourceTransformer">

                        <mainClass>
                            com.brandpdfpro.app.BrandPDFProApp
                        </mainClass>

                    </transformer>

                </transformers>

            </configuration>
        </execution>
    </executions>
</plugin>
Build JAR

Go to project root:

cd D:\BrandPDFPro_V1\BrandPDFPro

Build:

mvn clean package

Verify:

dir target

Expected:

brandpdfpro-1.0.0.jar
original-brandpdfpro-1.0.0.jar

Fat JAR should be several MB.

Example:

13 MB
Create Application Icon

Create:

assets
└── brandpdfpro.ico

Use a real Windows .ico file.

Recommended:

256x256 ICO
Generate App Image

Create release folder:

mkdir release

Generate:

jpackage ^
 --type app-image ^
 --name BrandPDFPro ^
 --input target ^
 --main-jar brandpdfpro-1.0.0.jar ^
 --main-class com.brandpdfpro.app.BrandPDFProApp ^
 --icon assets\brandpdfpro.ico ^
 --dest release
Output

Generated structure:

release
└── BrandPDFPro
    ├── BrandPDFPro.exe
    ├── app
    └── runtime
Test EXE

Run:

release\BrandPDFPro\BrandPDFPro.exe

or double-click.

Verify:

✅ Application Opens
✅ Header/Footer Works
✅ Batch Processing Works
✅ Settings Persist
Optional - Generate Installer

Install WiX Toolset.

Verify:

candle.exe -?
light.exe -?

Then:

jpackage ^
 --type exe ^
 --name BrandPDFPro ^
 --input target ^
 --main-jar brandpdfpro-1.0.0.jar ^
 --main-class com.brandpdfpro.app.BrandPDFProApp ^
 --icon assets\brandpdfpro.ico ^
 --dest release

Output:

BrandPDFPro-1.0.0.exe

Windows installer.
